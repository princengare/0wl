data migration

0wl keeps historical browsing data local and migrates settings forward without clearing extension storage.

current storage

IndexedDB focus_tracker v1 sessions source of truth daily_usage aggregate block_attempts minute buckets browser.storage.local settings/runtime/lifecycle

migration behavior

  • Existing blocked sites migrate to Always active.
  • Existing time limits migrate to Always active.
  • Malformed settings are repaired without clearing valid records.
  • Stale runtime sessions are invalidated without counting Firefox downtime.
  • Dynamic block and time-limit rules are rebuilt from settings on startup.

future schema changes

  • Increment the IndexedDB version deliberately.
  • Add forward migrations instead of deleting stores.
  • Preserve completed sessions and blocked/time-limit configuration.
  • Add tests for old-to-new data shapes.
