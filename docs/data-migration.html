data migration

0wl keeps historical browsing data local and migrates settings forward without clearing extension storage.

current storage

IndexedDB focus_tracker v2 sessions source of truth daily_usage aggregate block_attempts minute buckets domain_transitions vision patterns browsing_intents friction prompts browser.storage.local settings/runtime/lifecycle/vision

migration behavior

  • Existing blocked sites migrate to Always active.
  • Existing time limits migrate to Always active.
  • Version 2 adds domain_transitions and browsing_intents.
  • Vision settings and category overrides are additive local-storage records.
  • Malformed settings are repaired without clearing valid records.
  • Stale runtime sessions are invalidated without counting Firefox downtime.
  • Dynamic block and time-limit rules are rebuilt from settings on startup.

browser-specific storage

Firefox, Chrome, Edge, Opera, and Safari each keep separate extension storage. Safari does not automatically import data from another browser. 0wl does not add cloud sync or account-based transfer.

data control in 0.1.4

Settings Data Control can show local counts, export a JSON backup, import backups by merge or confirmed replace, set history retention with Forever as the default, clear selected local data categories, and reset settings or all local data only after typed confirmation.

These actions operate only on local extension storage in the current browser. They do not add cloud sync, accounts, telemetry, or remote backup.
