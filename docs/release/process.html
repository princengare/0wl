release checklist

  1. Update package.json to the new version. WXT generates the extension manifest version from package metadata.
  2. Update CHANGELOG.md.
  3. Run npm run release:prepare.
  4. Review the generated WXT artifact in .output/.
  5. Sign the extension with npm run sign:firefox.
  6. Publish through AMO or host the signed self-distributed XPI.
  7. For Safari, run npm run safari:convert, then sign/archive the generated app wrapper in Xcode.
