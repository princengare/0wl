development workflow

Build and reload the Firefox extension locally without using a backend, localhost service, cloud account, or external API.

one command

npm run firefox:dev

This command runs an initial build, starts TypeScript watch, starts Vite watch, waits for the first watched build, then launches Firefox through Mozilla web-ext.

Watch builds keep existing files in dist so Firefox does not reload a half-empty extension folder.

manual loop

npm run build
npm run firefox

Manual loading should always use dist/manifest.json, not public/manifest.json.

data safety

  • Use the dedicated .web-ext-profile/ profile for development.
  • Do not commit profile data, IndexedDB snapshots, local exports, or signing secrets.
  • Build output can be recreated; extension storage should be treated as user data.
