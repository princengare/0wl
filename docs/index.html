local-first website focus tracker

0wl is a standalone local-first extension for tracking active website time, blocking distracting domains, scheduling blocks and limits, and reviewing terminal-style history charts, local behavioral insights, and scheduled friction. Version 0.1.3 adds the first local Vision layer, WXT builds for Firefox, Chrome, Edge, Opera, and Safari assets, and UI changes that make the extension more consistent with the 0wl aesthetic from one shared codebase. No accounts. No backend. No cloud sync.

0wl aesthetic note

bundled font Extension pages and this site use bundled JetBrains Mono with ss02, cv03, and slashed-zero features.
dashboard consistency Header tabs stay on one line, Settings uses the gear control, and Vision section titles are denser.
history alignment The weekly average line now sits at the computed chart value.

Current features

usage tracking Tracks active HTTP/HTTPS domains only while Firefox is focused and the system is active.
history charts Today and yesterday use hourly bars. This week uses calendar-week bars and an average line.
terminal ui controls Custom dropdowns, bracket checkboxes, one-line dashboard tabs, and bundled JetBrains Mono keep the UI consistent.
scheduled blocking Block websites always or only on selected local days and times, including overnight windows.
scheduled limits Apply daily time limits always or only during scheduled windows, with a temporary bypass page.
vision insights Local site categories, transitions, distraction pathways, recovery estimates, block outcomes, and recommendations.
local storage Uses IndexedDB and browser.storage.local. No telemetry, accounts, or cloud database.
browser-specific builds WXT builds Firefox, Chrome, Edge, Opera, and Safari web-extension assets from the shared source tree.

Roadmap note

0wl should eventually separate browser activity into attention buckets instead of treating everything as the same browsing time.

active browsing main focused HTTP/HTTPS tab while the user is active PiP media Picture-in-Picture media tracked separately from active browsing background media audio or video playing from non-active tabs idle visible visible page time while the user is idle browser unfocused browser pages open while another app has attention

Default direction: track PiP and background media separately, do not count them toward active browsing by default, and let users decide whether media exposure counts toward site limits.
