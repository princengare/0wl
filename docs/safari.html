safari support

0wl now builds Safari web-extension assets through WXT. Safari still requires Apple's Xcode app wrapper for local testing, signing, archiving, and distribution.

build assets

npm run build:safari

Output is generated at .output/safari-mv2/. Safari is intentionally not included in npm run build:all because the final Safari flow requires Xcode.

xcode wrapper

npm run safari:convert
npm run safari:open

The wrapper is generated under platforms/safari/xcode/. Use Xcode to run the app, enable the extension in Safari, and test the extension manually. Regenerating the wrapper can overwrite generated Xcode resources, so re-check local signing settings.

known caveats

  • Safari extension storage is separate from Firefox, Chrome, Edge, and Opera data.
  • Idle detection is partial and guarded at runtime.
  • Scheduled rules use alarms when Safari exposes the API.
  • Blocking, time-limit, and friction redirects require Safari DNR behavior testing.
  • Manual Safari testing is required before claiming full Safari parity.
